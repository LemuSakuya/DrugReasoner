药研智析 DrugReasoner
融合语言理解与符号关系推理的药物数据分析系统

一、系统定位
本软件面向药物关系智能分析场景，围绕药物-药物相互作用、药物-靶标亲和力预测、蛋白/靶标查询和可解释报告生成提供一体化原型。系统将本地结构化数据库、LLMDTA 预测模型、符号关系图和 LLM 智能助手组合起来，用于支持药物重定位、联合用药风险分析和药物研发教学演示。

二、主要功能
1. 药物信息查询
输入药物名称后，可查看药物编号、副作用、PubChem 化学特征、2D/3D 分子结构和关系图。

2. 药物-药物分析
输入两个药物名称后，系统会查询已知 DDI 关系；若数据库未收录，则调用本地预测逻辑输出协同或拮抗倾向，并提供符号关系图展示。

3. DTA 预测
输入药物名称与靶标名称后，系统会准备预测输入并调用 LLMDTA 模型，输出药物-靶标亲和力预测值。靶标名称建议使用 davis_prots.csv 的 prot_id。

4. DTA/DDI 对比
在同一窗口中比较药物-靶标亲和力预测与药物-药物符号关系预测，便于答辩演示多任务分析能力。

5. 智能助手
支持自然语言提问，例如：
- 查询 Aspirin 的药物信息
- Aspirin 和 Warfarin 的相互作用
- EGFR 相关靶标信息
- Aspirin 和 Warfarin 的相互作用并生成证据报告

三、首次使用
1. 运行 setup.ps1 安装依赖并启动程序。
2. 首次使用数据库时运行 setup.ps1 -InitDB 导入 data/drug_discovery_dump.sql。
3. 如果需要智能助手联网调用大模型，请复制 .env.example 为 .env，并填写 LLM_API_KEY、LLM_BASE_URL、LLM_MODEL。
4. 若本机已有 MySQL，可使用 setup.ps1 -SkipDocker 跳过 Docker。

四、输出文件
智能助手生成可解释报告后，会输出：
- Markdown 报告：reports/
- JSON 结构化结果：data/exports/json/

五、注意事项
本系统为科研训练与课程展示原型，输出结果应作为数据分析参考，不可直接替代临床诊疗或真实用药决策。
